# 开发计划生成规范

> 通用开发计划生成指导性文件，定义开发计划应该怎么生成的核心方法论

**版本：** v2.0  
**日期：** 2026-05-19  
**变更说明：** 取消"计划拆解"任务单，工作计划清单已明确定义所有任务

---

## 一、规范层级与定位

### 1.1 开发计划文档架构

AutoCraft 开发计划采用两层文档架构：

| 层级 | 文档类型 | 说明 | 示例 |
|------|---------|------|------|
| **总览级** | 项目总览文档 | 批次顺序、模块依赖、功能概览 | 开发计划总览.md |
| **计划级** | 工作计划清单 | 单个工作计划的任务单结构概要 | WP-M03-BE-DEV-001.md |

**文档关系**：
```
设计层（功能设计文档）
    ↓ 输入
开发计划总览（项目级）
    ↓ 指引
工作计划清单（计划级）
    ↓ 直接执行
程序代码 + 测试代码（产出物）
```

### 1.2 核心设计原则

| 原则 | 说明 |
|------|------|
| **清单预生成保障整体性** | 编号统一规划、命名规范一致、任务数量明确 |
| **无拆解任务** | 清单已明确定义所有任务，直接执行即可 |
| **AI作为智能开发者** | 输入设计文档原文而非预定义步骤 |
| **pytest闭环应对不确定性** | 失败→修复→重试，迭代修复是不可预测的 |

## 二、任务结构定义

### 2.1 任务编号规则（无拆解任务）

| 计划类型 | 任务编号范围 | 任务结构 |
|---------|-------------|---------|
| **INFRA** | 001~006 | 骨架→环境→数据库→初始化→启动→报告 |
| **BE-DEV** | 001~N-1 | 功能循环（每功能：测试代码→程序代码）→ 报告 |
| **BE-L2** | 001~N-1 | 环境准备 → 场景循环 → 报告 |
| **FE-DEV** | 001~N-1 | 功能循环（每功能：测试代码→程序代码）→ 报告 |
| **FE-L2** | 001~N-1 | 环境准备 → 场景循环 → 报告 |
| **L3-E2E** | 001~N-1 | 环境准备 → 流程循环（两层测试）→ 报告 |

### 2.2 任务结构示例

**BE-DEV 计划**：
```
001: F-001-测试代码生成 → BUILD
002: F-001-程序代码+pytest → BUILD
003: F-002-测试代码生成 → BUILD
004: F-002-程序代码+pytest → BUILD
...
N-1: BE-DEV报告 → DOC
```

**BE-L2 计划**：
```
001: 测试环境准备 → BUILD
002: 场景1测试 → BUILD
003: 场景2测试 → BUILD
...
N-1: BE-L2报告 → DOC
```

**INFRA 计划**：
```
001: 项目骨架搭建 → BUILD
002: 开发环境配置 → BUILD
003: 数据库代码生成 → BUILD
004: 数据库初始化 → EXECUTE
005: 启动脚本创建 → BUILD
006: 验收报告 → DOC
```

## 三、一致性保障机制

### 3.1 编号约束机制

工作计划清单提供编号约束，执行时必须遵守：

```markdown
## 任务单编号约束

**BE-DEV/FE-DEV计划**：
| 任务编号范围 | 任务类型 | 说明 |
|-------------|---------|------|
| 001~N-2 | BUILD | 功能循环（每功能2个任务）|
| N-1 | DOC | 开发报告 |

**BE-L2/FE-L2计划**：
| 任务编号范围 | 任务类型 | 说明 |
|-------------|---------|------|
| 001 | BUILD | 测试环境准备 |
| 002~N-2 | BUILD | 场景循环 |
| N-1 | DOC | 测试报告 |

**L3-E2E计划**：
| 任务编号范围 | 任务类型 | 说明 |
|-------------|---------|------|
| 001 | BUILD | 测试环境准备 |
| 002~N-2 | BUILD | 流程循环（两层测试）|
| N-1 | DOC | E2E报告 |
```

### 3.2 版本一致性检查

工作计划清单标注设计文档版本，启动时检查：

```markdown
## 设计文档版本

| 文档类型 | 当前版本 | 文档路径 |
|---------|---------|---------|
| API设计 | v1.2 | docs/design/API设计-v1.2.md |
| 数据库设计 | v1.0 | docs/design/数据库设计-v1.0.md |

**版本检查规则**：
启动时读取设计文档版本号，与清单标注版本对比，不一致则报错暂停。
```

## 四、生成流程

### 4.1 制定阶段生成流程

```
开发计划制定阶段：
    │
    ├─ 第1批：INFRA + 第一批模块
    │     ├─ 生成开发计划总览
    │     ├─ 生成工作计划清单
    │     └─ 审阅 → Git提交
    │
    ├─ 第2批：第二批模块
    │     ├─ 生成工作计划清单
    │     └─ 审阅 → Git提交
    │
    └─ 完成：全部清单
```

### 4.2 执行阶段流程

```
执行阶段：
    │
    ├─ 读取工作计划清单
    ├─ 版本一致性检查
    │
    ├─ 循环执行任务单（001~N-1）
    │     ├─ 读取设计文档原文
    │     ├─ AI自主决策实现方式
    │     ├─ pytest/vitest闭环迭代
    │     └─ 验证子代理业务验证
    │
    └─ 完成：更新计划状态
```
