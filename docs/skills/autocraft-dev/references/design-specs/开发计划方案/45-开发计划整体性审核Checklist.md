# 开发计划整体性审核 Checklist

> 开发计划跨文档一致性审核，对总览+全部清单整体审核，确保上下文一致、逻辑无冲突、无重复冗余

**版本：** v1.0
**日期：** 2026-04-16
**适用文档：**
- 41-开发计划总览生成规范
- 42-工作计划清单生成规范（按项目实际份数）
- 43-开发计划总览审核Checklist
- 44-工作计划清单审核Checklist
**审核时机：** 总览审核通过 + 全部清单批量审核通过后，进行整体性审核

---

## 使用说明

**审核维度**：

| 维度 | 说明 | 扣分权重 |
|------|------|---------|
| **上下文一致性** | 总览与各清单数据是否匹配 | 10分/项 |
| **逻辑冲突** | 各清单编号规则/依赖关系是否冲突 | 10分/项 |
| **重复冗余** | 总览与清单是否有重复描述 | 5分/项 |
| **引用完整性** | 清单引用测试方案/设计文档是否正确 | 5分/项 |
| **批次依赖一致性** | 批次顺序在各清单依赖关系是否一致 | 10分/项 |

**评分计算**：
```
总分 = 100 - (一致性缺失数 × 10) - (逻辑冲突数 × 10) - (重复冗余数 × 5) - (引用缺失数 × 5)
```

**评分等级**：
- ≥90分：通过，整体定稿，进入开发执行
- 70-89分：需修改，修复一致性/逻辑冲突后重新审核
- <70分：不通过，重新设计开发计划体系

## 七、快速审核流程

```
Step1: 检查X01-X10（上下文一致性）
    ├─ 对照总览与各清单数量统计（实际份数、API数、场景数）
    ├─ 对照总览与各清单模块信息（名称、功能ID、批次）
    └─ 统计不一致数量 → 计算扣分

Step2: 检查L01-L15（逻辑冲突）
    ├─ 对照各模块编号规则一致性
    ├─ 对照各模块依赖关系一致性
    └─ 对照各模块批次顺序一致性
    └─ 统计冲突数量 → 计算扣分

Step3: 检查R01-R10（重复冗余）
    ├─ 检查验收标准引用而非重复定义
    ├─ 检查编号规则引用而非重复定义
    └─ 统计重复数量 → 计算扣分

Step4: 检查Y01-Y10（引用完整性）
    ├─ 检查总览引用设计文档完整性
    └─ 检查各清单引用测试方案完整性
    └─ 统计缺失数量 → 计算扣分

Step5: 检查D01-D10（批次依赖一致性）
    ├─ 检查批次内依赖关系一致
    └─ 检查模块闭环依赖顺序一致
    └─ 统计不一致数量 → 计算扣分

Step6: 计算总分 → 判断结论

Step7: 填写缺失项清单 → 提供修复建议

Step8: 签字确认
```
