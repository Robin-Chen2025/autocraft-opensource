# 开发计划审核通用提示词

**版本：** v2.0
**日期：** 2026-04-16

---

## 输入参数

| 参数 | 说明 | 示例 |
|------|------|------|
| 待审核文档 | 开发计划文档路径 | `docs/development-plans/开发计划总览.md` |
| Checklist文件 | 对应审核Checklist | `43-开发计划总览审核Checklist.md` 或 `44-工作计划清单审核Checklist.md` |
| 对照文档清单 | C系列内容审核所需 | API设计+系统功能设计+测试方案 |

---

## 执行流程

严格按照 **Checklist文件** 逐项检查：

1. **E系列**：格式完整性检查
2. **C系列**：对照设计文档验证一致性
3. **W系列**：细节完整性检查

评分计算：
```
总分 = 100 - (阻断项缺失数 × 10) - (内容审核缺失数 × 5) - (警告项缺失数 × 2)
```

---

## 输出要求

- 输出审核报告（评分 + 结论 + 缺失项 + 修复建议）
- 输出路径：`docs/development-plans/{文档名称}-审核报告-v{版本号}.md`
- 结论判断：≥80分通过，<80分不通过
