## 特别注意事项（必须理解）

> **测试方案已预生成，直接引用路径！**

| ❌ 错误理解 | ✅ 正确理解 |
|-----------|-----------|
| 清单需要生成测试方案任务 | 基本信息"测试方案引用"列标注路径即可 |
| BE-DEV有固定任务002/003 | BE-DEV只有001拆解，002~N都是功能循环 |
| 任务编号预定义 | 任务编号动态计算：1拆解 + 功能数×2 + 1报告 |

**编号规则（严格遵守）**：
- BE-DEV/FE-DEV：001拆解 → 002~N-1功能循环 → N报告
- BE-L2/FE-L2：001拆解 → 002环境 → 003~N-1场景循环 → N报告
- L3-E2E：001拆解 → 002环境 → 003~N-1流程循环（两层）→ N报告

---

## 执行流程

严格按照 **42-工作计划清单生成规范.md** 第8章执行流程执行。

生成7个章节（详见42规范第2章）：
1. 基本信息（含测试方案引用）
2. 输入文档（含版本号）
3. 功能模块概览
4. 任务单结构表格（核心章节）
5. 验收标准
6. 编号约束规则
7. 预计工时

---

## 输出要求

- 总长度：~120行
- 对照 **44-工作计划清单审核Checklist.md** 确保所有E+C系列检查项满足
- C系列内容审核：API数量、功能ID与设计文档一致
- 输出路径：`docs/development-plans/{模块}/WP-{模块}-{类型}-001.md`
