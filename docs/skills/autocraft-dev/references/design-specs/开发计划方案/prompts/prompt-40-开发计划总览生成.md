## 特别注意事项（必须理解）

> **测试方案已预生成，开发计划直接引用！**

| ❌ 错误理解 | ✅ 正确理解 |
|-----------|-----------|
| 开发计划需要生成测试方案任务 | 测试方案已在制定阶段预生成，不需要生成任务 |
| BE-DEV有固定任务002/003 | BE-DEV只有001拆解，002~N都是功能循环 |
| API数量需自己估算 | API数量从API设计文档接口清单统计 |
| 场景数需自己估算 | 场景数从测试方案文档统计 |

**批次顺序核心原则**：M-03必须在第3批（依赖M-02/M-04/M-06）

## 执行流程

严格按照 **41-开发计划总览生成规范.md** 第9章执行流程执行。

生成7个章节（详见41规范第2章）：
1. 项目概述
2. 批次顺序与执行策略
3. 模块依赖矩阵
4. 功能模块概览（API数+场景数统计）
5. 工作计划数量统计
6. 验收标准汇总
7. 执行监控与阻塞处理

- C系列内容审核：API数量、场景数与设计文档一致
- 输出路径：`docs/development-plans/开发计划总览.md`
