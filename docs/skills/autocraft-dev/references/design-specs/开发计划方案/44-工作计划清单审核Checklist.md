# 工作计划清单审核Checklist

> 工作计划清单审核标准，用于审核人员快速检查清单文档完整性

**版本：** v1.1
**日期：** 2026-04-16
**指导性文件：**
- 40-开发计划生成规范.md
- 42-工作计划清单生成规范.md
**审核对象：**
- 工作计划清单.md（按项目实际份数）
**作者：** AutoCraft 团队

---

## 一、审核说明

### 1.1 审核目标

确认工作计划清单包含完整的任务单结构表格、编号约束、版本标注、验收标准引用。

### 1.2 评分计算

```
总分 = 100 - (阻断项数量 × 10) - (内容审核数量 × 5) - (警告项数量 × 2)

通过标准：总分 ≥ 80分
```

> **C系列（内容审核）**：检查与设计文档、测试方案文档的一致性，每项扣5分

---

## 二、阻断项（E系列，每项扣10分）

| 检查项ID | 检查内容 | 通过条件 | 检查方式 |
|---------|----------|---------|---------|
| **E01** | 基本信息完整 | 包含计划ID、模块、类型、批次、依赖、测试方案引用6项 | 查找"基本信息"表格 |
| **E02** | 输入文档版本标注 | 第2章包含版本号列 | 查找"当前版本"列 |
| **E03** | 任务单结构表格完整 | 第4章包含编号、内容、类型、验证分类、功能归属、产出物6列 | 查找表格列数=6 |
| **E04** | 编号约束定义存在 | 第6章包含约束表格 | 查找"编号约束规则"表格 |
| **E05** | 验收标准引用正确 | 第5章引用测试方案总纲章节 | 查找测试方案总纲引用（"详见《...测试方案》"格式） |
| **E06** | 功能归属与系统功能设计一致 | 功能ID与系统功能设计文档匹配 | 对照系统功能设计文档 |

---

## 三、警告项（W系列，每项扣2分）

| 检查项ID | 检查内容 | 通过条件 | 检查方式 |
|---------|----------|---------|---------|
| **W01** | 输入文档路径正确 | 文档路径存在 | 文件系统检查 |
| **W02** | 任务编号连续性 | 编号从001开始连续 | 检查编号连续性 |
| **W03** | 产出物路径符合规范 | 产出物路径符合项目结构 | 查找"backend/"、"frontend/"前缀 |
| **W04** | 预计工时标注 | 第7章包含工时表格 | 查找"预计工时"表格 |
| **W05** | 文档版本号标注 | 第1章包含版本号 | 查找"版本："行 |

---

## 四、内容审核项（C系列，每项扣5分）

> 检查与设计文档、测试方案文档的一致性

| 检查项ID | 检查内容 | 通过条件 | 检查方式 |
|---------|----------|---------|---------|
| **C01** | API数量与模块API设计一致 | 第3章API数量与API设计文档对应模块章节一致 | 对照API设计文档模块章节 |
| **C02** | 功能ID范围与系统功能设计一致 | 第3章功能ID范围与系统功能设计文档模块章节一致 | 对照系统功能设计文档 |
| **C03** | L1测试场景数与测试方案一致 | 第3章L1场景数与L1测试方案对应模块一致（BE-DEV/FE-DEV）| 对照L1测试方案 |
| **C04** | L2测试场景数与测试方案一致 | 第3章L2场景数与L2测试方案对应模块一致（BE-L2/FE-L2）| 对照L2测试方案 |
| **C05** | 任务单数量与功能数量匹配 | 任务单数量 = 固定任务 + 功能数×2 + 报告（BE-DEV）| 计算验证 |
| **C06** | BE-L2/FE-L2任务粒度符合规范 | 003~N-1任务数 = 测试方案场景数（每场景1个任务）| 对照L2测试方案场景清单 |
| **C07** | 跨功能场景功能归属正确 | 跨功能场景功能归属标注为"跨功能"或"-"，不归属单一功能 | 检查场景API串联是否跨功能 |

## 五、不同计划类型特殊检查项

### 4.1 BE-DEV/FE-DEV计划

| 检查项ID | 检查内容 | 通过条件 |
|---------|----------|---------|
| **E07-DEV** | 功能循环任务成对出现 | 每功能2个任务（测试代码→程序代码+pytest闭环）|
| **E08-DEV** | 不包含方案设计任务 | 无"方案设计"、"设计方案"等DOC类任务单 |
| **W06-DEV** | 测试方案引用存在 | 基本信息"测试方案引用"列存在且路径正确 |

### 4.2 BE-L2/FE-L2计划

| 检查项ID | 检查内容 | 通过条件 |
|---------|----------|---------|
| **E07-L2** | 拆解任务单存在 | 001=拆解任务单 |
| **E08-L2** | 测试环境任务存在 | 002=测试环境准备 |
| **E09-L2** | 任务粒度符合规范 | 003~N-1任务数 = 测试方案场景数（每场景1个任务）|
| **W06-L2** | 场景数量与测试方案一致 | 任务单数量与L2测试方案场景数匹配 |

### 4.3 L3-E2E计划

| 检查项ID | 检查内容 | 通过条件 |
|---------|----------|---------|
| **E07-L3** | 两层测试任务成对出现 | 每流程2个任务（两层测试）|
| **E08-L3** | 测试范围定义明确 | 基本信息"测试范围"列存在 |

### 4.4 INFRA计划（基础建设）

| 检查项ID | 检查内容 | 通过条件 | 检查方式 |
|---------|----------|---------|---------|
| **E07-INFRA** | 数据库代码生成任务拆分规则正确 | 按依赖层级拆分（A=Level 0, B=Level 1...）| 查找拆分规则章节 |
| **E08-INFRA** | 数据库初始化任务为EXECUTE类型 | 数据库初始化任务类型=EXECUTE，依赖数据库代码生成全部产出 | 查找"EXECUTE"类型标注 |
| **E09-INFRA** | 依赖层级分析存在 | 包含依赖层级分析或表分组定义 | 查找"Level 0~N"或"依赖层级" |
| **W06-INFRA** | 拆分必要性说明存在 | 包含"拆分必要性"、"拆分条件"说明 | 查找"必要性"文字 |
| **W07-INFRA** | 表依赖关系与数据库设计一致 | Level 0表为无外键依赖的根表 | 对照数据库设计文档 |

### 4.5 各计划类型任务单数量规则

| 计划类型 | 任务单数量规则 | 说明 |
|---------|--------------|------|
| **INFRA** | 001 + 002~003 + 数据库代码(A~N) + 数据库初始化 + 启动脚本 + 验收报告 | 数据库代码按依赖层级拆分为A/B/C/D/E... |
| **BE-DEV** | 001 + (功能数×2) + 报告N | 每功能2个任务（测试→程序） |
| **BE-L2** | 001 + 002 + 场景数 + 报告N | 001拆解，002环境，场景循环 |
| **FE-DEV** | 001 + (功能数×2) + 报告N | 同BE-DEV |
| **FE-L2** | 001 + 002 + 场景数 + 报告N | 同BE-L2 |
| **L3-E2E** | 001 + 002 + (流程数×2) + 报告N | 每流程2个任务（两层测试）|

## 六、审核流程

```
第1步：读取工作计划清单.md
第2步：检查计划类型（BE-DEV/BE-L2/FE-DEV/FE-L2/L3-E2E）
第3步：检查通用阻断项E01-E06
第4步：检查计划类型特殊阻断项（E07/E08）
第5步：检查内容审核项C01-C05
第6步：检查警告项W01-W05
第7步：计算总分
第8步：判定通过/不通过
第9步：生成审核报告
```
