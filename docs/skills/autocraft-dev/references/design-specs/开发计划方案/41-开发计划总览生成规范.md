## 一、规范定位

### 1.1 总览定位

开发计划总览是**项目级管理文档**，包含：
- 批次顺序与执行策略
- 模块依赖矩阵
- L3-E2E增量测试节点
- 功能模块概览
- 工作计划数量统计

**用户审阅入口**：总览让用户掌握整体规划，确认批次顺序、依赖关系、增量测试节点。

### 1.2 与工作计划清单关系

| 文档 | 粒度 | 内容 | 用户审阅 |
|------|------|------|---------|
| **开发计划总览** | 项目级 | 批次顺序、模块依赖、功能概览 | ✅ 审阅整体规划 |
| **工作计划清单** | 计划级 | 单个计划的任务单结构概要 | ✅ 审阅具体计划 |

**生成顺序**：总览先生成 → 工作计划清单按总览指引生成

## 二、总览章节结构规范

### 2.1 总览章节结构（7章节）

| 章节 | 内容 | 长度预估 | 对用户的价值 |
|------|------|---------|-------------|
| **1. 项目概述** | 项目名称、版本、开发目标 | ~10行 | 项目定位 |
| **2. 批次顺序与执行策略** | 4批模块顺序、并行策略、L3-E2E节点 | ~40行 | **核心章节** |
| **3. 模块依赖矩阵** | 模块调用依赖、前置条件 | ~30行 | 开发顺序依据 |
| **4. 功能模块概览** | 5模块功能列表、API数、场景数 | ~50行 | 覆盖范围确认 |
| **5. 工作计划数量统计** | 按项目实际计划清单数量表格 | ~20行 | 规划规模 |
| **6. 验收标准汇总** | 引用测试方案总纲验收标准 | ~20行 | 验收依据 |
| **7. 执行监控与阻塞处理** | 批次完成标准、阻塞处理规则 | ~30行 | 风险管控 |

**总长度**：~200行（紧凑版，用户快速审阅）

### 3.2 执行策略

```markdown
## 执行策略

### 并行策略
- 同批次模块可并行开发（如M-01与M-06）
- 模块内部闭环必须顺序执行（BE-DEV → BE-L2 → FE-DEV → FE-L2）

### 模块闭环执行顺序
每个模块按以下顺序执行（4计划闭环）：
    ├─ WP-{模块}-BE-DEV-001（后端开发 + L1 pytest闭环）
    ├─ WP-{模块}-BE-L2-001（后端L2集成测试）
    ├─ WP-{模块}-FE-DEV-001（前端开发 + L1 vitest闭环）
    └─ WP-{模块}-FE-L2-001（前端L2集成测试）

### L3-E2E增量节点
| 节点 | 触发时机 | 测试范围 | 验收标准 |
|------|---------|---------|---------|
| L3-E2E-001 | 第1批模块闭环完成 | 项目创建 + 配置链查询 | Playwright两层100% |
| L3-E2E-002 | 第2批模块闭环完成 | 项目 + 计划 + 问题单 | Playwright两层100% |
| L3-E2E-003 | 第3批模块闭环完成 | FlowTicket完整执行 | Playwright两层100% |
| L3-E2E-Final | 全模块完成 | 全流程回归 + 边缘场景 | Playwright两层100% |
```

## 八、第七章：执行监控与阻塞处理规范

### 8.1 批次完成标准

| 批次 | 完成标准 |
|------|---------|
| 第1批 | M-01/M-06 BE-L2 pytest通过 + FE-L2 vitest通过 + L3-E2E-001通过 |
| 第2批 | M-02/M-04 BE-L2 pytest通过 + FE-L2 vitest通过 + L3-E2E-002通过 |
| 第3批 | M-03 BE-L2 pytest通过 + FE-L2 vitest通过 + L3-E2E-003通过 |
| 第4批 | L3-E2E-Final全部场景通过 |

### 8.2 阻塞处理规则

| 阻塞类型 | 处理方式 |
|---------|---------|
| 单模块阻塞 | 模块内创建问题单，模块暂停，其他模块继续 |
| 集成阻塞 | L3-E2E失败，创建问题单，相关模块暂停 |
| 批次阻塞 | 批次内多个模块阻塞，整批次暂停 |
