# 测试方案整体性审核 Checklist — V2

**版本：** v2.0
**日期：** 2026-05-13
**适用文档：** 测试方案总纲 + L1规范 + L2方案 + L3方案（整体审核）

---

## X系列：上下文一致性（每项5分）

| ID | 检查内容 | 通过标准 |
|----|---------|---------|
| X01 | 总纲层级定义与各方案一致 | L1条件触发+L2后端+L3前端梯度 |
| X02 | 总纲废弃层级说明与实际一致 | 前端L1/L2已废弃 |
| X03 | 各方案引用的总纲版本一致 | 版本号匹配 |
| X04 | 角色分离定义跨文档一致 | BUILD-TEST/TEST-RUN/FIX-CODE |

## L系列：逻辑一致性（每项5分）

| ID | 检查内容 | 通过标准 |
|----|---------|---------|
| L01 | 验收标准无矛盾 | 各层通过率、覆盖要求无冲突 |
| L02 | 执行顺序无矛盾 | BE-DEV→BE-L2→FE-DEV→L3-E2E |
| L03 | AI服务策略一致 | 各方案AI调用+超时处理一致 |
| L04 | 断言要求一致 | 各方案预设数据+精确验证原则一致 |

## R系列：重复冗余（每项5分）

| ID | 检查内容 | 通过标准 |
|----|---------|---------|
| R01 | 总纲与各方案无重复展开 | 总纲摘要，各方案展开 |
| R02 | L1/L2/L3场景无重复 | 各层职责边界清晰 |

## D系列：数据依赖（每项5分）

| ID | 检查内容 | 通过标准 |
|----|---------|---------|
| D01 | L2预设数据与数据库设计一致 | 表名、字段名匹配 |
| D02 | L3预设数据与L2共用 | 同一数据源 |
| D03 | AI服务超时跟生产配置一致 | 引用config.py |

**评分**：总分 = 100 - (X缺失 × 5) - (L缺失 × 5) - (R缺失 × 5) - (D缺失 × 5)
