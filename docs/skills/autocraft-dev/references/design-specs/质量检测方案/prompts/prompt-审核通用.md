# 测试方案审核通用提示词

**版本：** v2.0
**日期：** 2026-04-15

---

## 输入参数

| 参数 | 说明 | 示例 |
|------|------|------|
| 待审核文档 | 测试方案文档路径 | `docs/test-plans/{plan_id}-test-plan-overview.md` |
| Checklist文件 | 对应审核Checklist | `质量检测方案/25-总纲审核Checklist.md` |
| 对照文档清单 | C系列内容审核所需（按Checklist定义）| API设计+业务流程+数据库+PRD |

---

## 执行流程

严格按照 **Checklist文件** 逐项检查：

1. **E系列**：逐项检查格式完整性（按Checklist检查方式执行）
2. **C系列**（如有）：对照设计文档模拟验证（按Checklist对照文档清单）
3. **W系列**：逐项检查细节完整性

评分计算（按Checklist规则）：
```
总分 = 100 - (阻断项缺失数 × 10) - (内容审核缺失数 × 5) - (警告项缺失数 × 2)
```

---

## 输出要求

- 输出审核报告（评分 + 结论 + 缺失项清单 + 修复建议）
- 输出路径：`docs/test-plans/{plan_id}-审核报告-{文档类型}-v{版本号}.md`
- 结论判断（按Checklist规则）：≥90分通过，70-89分需修改，<70分不通过

---

## 对照文档清单（按Checklist类型）

| Checklist | 对照文档 |
|-----------|---------|
| 25-总纲 | 无需对照文档 |
| 26-BE-L2 | API设计 + 业务流程 + 数据库设计 + PRD |
| 27-FE-L2 | UI设计 + 组件规范 + 业务流程 + PRD |
| 28-L3 | 业务流程 + PRD + UI设计 + BE-L2方案 + FE-L2方案 |
| 29-整体性 | 总纲 + BE-L2 + FE-L2 + L3 |
