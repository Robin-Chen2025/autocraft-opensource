## 特别注意事项（必须理解）

> **总纲不是传统测试方案！**

| ❌ 错误理解 | ✅ 正确理解 |
|-----------|-----------|
| 第2章是测试用例清单表格 | 第5章是pytest闭环机制定义 |
| L1测试需要独立文档 | L1 pytest闭环在执行子代理内部 |
| 包含详细测试步骤 | 总纲是精简版，不展开具体场景 |
| 验收标准是功能通过率 | 验收标准是pytest通过率100% + 覆盖率≥80% |

**第2章"L1测试执行规范"必须包含**：
- pytest闭环机制（执行位置、修复次数上限）
- **L1测试场景清单表格**（模块→测试代码文件→场景数量）
- 验收标准表格（pytest通过率、覆盖率阈值、lint检查）
- Guardian检查项E4-E7（验证quality_check结果）
- execution_log.quality_check结构示例

## 执行流程

严格按照 **21-测试方案总纲生成规范.md** 第4章执行流程执行（8个Step）。

生成6个章节（详见21规范第2章）：

| 章节 | 内容 |
|------|------|
| 第1章 测试范围总览 | L1/L2/L3三层定义 + 任务类型分类（A/B/DOC）|
| **第2章 L1测试执行规范** | pytest闭环机制 + 验收标准 + Guardian检查项（**核心前置**）|
| 第3章 L2/L3测试策略摘要 | 职责分工 + 数据隔离 + 场景划分规则 |
| 第4章 场景覆盖矩阵 | BE-L2/FE-L2/L3场景数量预估表格 |
| 第5章 验收标准汇总 | 三层验收标准表格 + Guardian Block/Warning |
| 第6章 执行顺序依赖 | 执行流程图 + 数据依赖图 |
