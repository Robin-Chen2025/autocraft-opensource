# BE-L2审核 Checklist — V2

**版本：** v2.0
**日期：** 2026-05-13
**适用文档：** BE-L2测试方案

---

## E系列：阻断项（每项10分）

| ID | 检查内容 | 通过标准 |
|----|---------|---------|
| E01 | 5章节结构完整 | 测试范围/环境/场景清单/数据准备/验收标准 |
| E02 | 测试API清单明确 | 每个API列出，标注新增/已有 |
| E03 | 真实数据库要求 | 明确禁止mock数据库 |
| E04 | 测试场景清单存在 | 每个用例有预设数据+操作+预期结果 |
| E05 | 预设数据完整 | 明确字段值，不是"插入若干数据" |
| E06 | 预期结果精确 | 不能只有状态码，必须有数据内容验证 |
| E07 | AI服务策略 | 真实调用+超时=SKIP |
| E08 | 数据初始化脚本 | 能直接执行 |
| E09 | 验收标准明确 | pytest通过率100%+断言报告+数据落库验证 |

## C系列：内容审核（每项5分）

| ID | 检查内容 | 对照文档 |
|----|---------|---------|
| C01 | API覆盖完整 | 对照API设计文档 |
| C02 | 参数验证场景覆盖 | 对照API错误响应定义 |
| C03 | 数据边界验证 | 对照数据库设计字段约束 |
| C04 | AI服务场景覆盖 | 正向+异常+超时 |
| C05 | 写操作有数据落库验证 | 对照数据库设计 |

## W系列：警告项（每项2分）

| ID | 检查内容 | 通过标准 |
|----|---------|---------|
| W01 | 场景命名规范 | S-NORMAL/S-ERROR/S-EDGE |
| W02 | 数据隔离策略 | 不同场景不同ID |
| W03 | 断言报告格式 | 每个用例断言内容可输出 |
| W04 | 超时配置 | 跟生产配置走 |

**评分**：总分 = 100 - (E缺失 × 10) - (C缺失 × 5) - (W缺失 × 2)
