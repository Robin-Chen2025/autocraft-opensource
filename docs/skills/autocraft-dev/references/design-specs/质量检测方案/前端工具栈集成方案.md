## 一、工具栈定位

> **核心设计**：代码质量检查在执行子代理内部 vitest 闭环执行，Guardian 节点3只验证 quality_check 结果字段

AutoCraft 前端是 Vue 3 + TypeScript + Element Plus + Vite，已有基础工具配置。

| 工具 | 检查类型 | 集成位置 | 执行时机 | Guardian处理 |
|------|----------|----------|----------|-------------|
| **ESLint** | Lint 检查 | 执行子代理内部（vitest闭环修复阶段）| vitest失败后修复阶段 | 验证 quality_check.eslint_errors=0 |
| **Prettier** | Format 格式化 | 执行子代理内部（vitest闭环修复阶段）| vitest失败后修复阶段（自动）| 不验证（自动修复）|
| **TypeScript** | 类型检查 | 执行子代理内部（vitest闭环修复阶段）| vitest失败后修复阶段 | 验证 quality_check.typescript_errors=0 |
| **Vitest** | 测试 + 覆盖率 | 执行子代理内部（vitest闭环）| 代码生成后立即执行 | 验证 quality_check.vitest_passed=true + coverage≥70% |

**层级划分说明**：
- **L1（vitest闭环内部）**：Vitest + ESLint + Prettier + TypeScript

## 二、ESLint 详细方案

### 2.1 定位

**集成位置**: 执行子代理内部 vitest 闭环（修复阶段）

**执行时机**: vitest 失败后针对性修复阶段

> **Guardian 节点3职责**：验证 `quality_check.eslint_errors=0`，不执行 ESLint 检查

### 2.2 ESLint 配置方案（Vue 3 + TypeScript + ESLint 9.x）

**文件位置**: `frontend/eslint.config.js`

```javascript
// eslint.config.js
import js from '@eslint/js'
import typescript from 'typescript-eslint'
import vue from 'eslint-plugin-vue'
import prettier from 'eslint-plugin-prettier'

export default [
  // 基础 JS 规则
  js.configs.recommended,
  
  // TypeScript 规则
  ...typescript.configs.recommended,
  
  // Vue 规则
  ...vue.configs['vue3-recommended'],
  
  // Prettier 集成
  {
    plugins: {
      prettier: prettier,
    },
    rules: {
      'prettier/prettier': 'error',
    },
  },
  
  // 自定义规则
  {
    rules: {
      // TypeScript
      '@typescript-eslint/no-unused-vars': ['error', { argsIgnorePattern: '^_' }],
      '@typescript-eslint/explicit-function-return-type': 'off',
      '@typescript-eslint/no-explicit-any': 'warn',
      
      // Vue
      'vue/multi-word-component-names': 'off',  // 允许单词组件名
      'vue/no-v-html': 'warn',  // v-html 安全警告
      'vue/require-default-prop': 'off',  // 允许无默认值的 prop
      'vue/attribute-hyphenation': ['error', 'always'],  // 属性命名规范
      
      // 代码风格
      'no-console': 'warn',
      'no-debugger': 'error',
      'prefer-const': 'error',
      'no-var': 'error',
    },
  },
  
  // 排除文件
  {
    ignores: [
      'dist/**',
      'node_modules/**',
      '*.d.ts',
      'vite.config.ts',
    ],
  },
]
```

### 2.3 ESLint 执行命令

```bash
# 检查所有文件
eslint src/

# 检查并自动修复
eslint src/ --fix

# 仅检查错误（忽略警告）
eslint src/ --quiet

# 输出 JSON（供 Guardian 解析）
eslint src/ --format json

# 检查特定文件类型
eslint src/**/*.vue src/**/*.ts
```

### 2.4 ESLint 阻断级别设计（vitest闭环内部）

| 问题级别 | ESLint 规则 | vitest闭环处理 | 说明 |
|----------|-------------|----------------|------|
| **Error** | error 级别规则 | **自动修复** | vitest闭环内针对性修复 |
| **Warning** | warn 级别规则 | 仅提示 | 不阻断vitest闭环 |
| **Format** | prettier/prettier | **自动格式化** | 执行子代理内部自动处理 |

### 2.5 vitest闭环修复流程

```
执行子代理内部 vitest 闭环：
    │
    ├─ Crush生成Vue/TS代码
    │
    ├─ vitest执行
    │     ├─ vitest通过 + coverage≥70% → 继续
    │     └─ vitest失败 → 进入修复阶段
    │
    ├─ 修复阶段（针对性修复）：
    │     ├─ ESLint检查 → 发现error → `eslint --fix`自动修复
    │     ├─ Prettier格式化 → `prettier --write`自动格式化
    │     ├─ TypeScript类型检查 → 针对性修复
    │     ├─ 重新vitest → 循环最多3次
    │     └─ 超限 → 返回超限结果 → FlowTicket创建问题单
    │
    └─ vitest通过 → 产出quality_check字段
          ├─ vitest_passed: true
          ├─ coverage_percent: 75
          ├─ eslint_errors: 0
          ├─ eslint_warnings: 2
          └─ typescript_errors: 0
```

> **Guardian节点3验证**：检查 `quality_check.eslint_errors=0`，不为0则返工节点2

## 五、Vitest 详细方案

### 5.1 定位

**集成位置**: 执行子代理内部（前端 pytest 闭环等效）

**功能**: 单元测试 + 覆盖率检测

### 5.2 Vitest 配置方案

**文件位置**: `frontend/vitest.config.ts`

```typescript
// vitest.config.ts
import { defineConfig } from 'vitest/config'
import vue from '@vitejs/plugin-vue'
import { resolve } from 'path'

export default defineConfig({
  plugins: [vue()],
  test: {
    // 测试环境
    environment: 'jsdom',
    globals: true,
    
    // 测试文件匹配
    include: ['tests/**/*.test.ts', 'tests/**/*.spec.ts'],
    exclude: ['node_modules', 'dist'],
    
    // 覆盖率配置
    coverage: {
      provider: 'v8',
      reporter: ['text', 'html', 'json'],
      reportsDirectory: './tests/reports/coverage',
      
      // 阈值设置
      thresholds: {
        lines: 70,
        functions: 70,
        branches: 70,
        statements: 70,
        'autoUpdate': false,
      },
      
      // 覆盖范围
      include: ['src/**/*.ts', 'src/**/*.vue'],
      exclude: [
        'src/**/*.d.ts',
        'src/types/**',
        'src/router/**',
      ],
    },
  },
  resolve: {
    alias: {
      '@': resolve(__dirname, 'src'),
    },
  },
})
```

### 5.3 Vitest 执行命令

```bash
# 运行测试
vitest run

# 运行测试 + 覆盖率
vitest run --coverage

# 监听模式（开发时）
vitest watch

# 仅覆盖率检查
vitest run --coverage --reporter=json

# UI 界面（可选）
vitest --ui
```

### 5.4 Vitest 覆盖率阈值

| 任务类型 | 覆盖率阈值 | 说明 |
|----------|------------|------|
| **前端任务**（FE-DEV） | ≥ 70% | 前端测试难度更高，阈值略低 |
| **前端核心组件** | ≥ 80% | 公共组件提高要求 |
| **前端 L2 测试** | ≥ 90% | 集成测试覆盖率 |

### 5.5 Vitest pytest 闭环等效逻辑

> **vitest闭环详细流程**：见第七章7.1节

**闭环核心逻辑**：
```
Crush生成Vue/TS代码 → vitest执行（覆盖率）→ 
  【失败】进入修复阶段 → 循环最多3次 → 
    【超限】创建问题单
    【成功】产出quality_check
```

**产出字段**：vitest_passed、coverage_percent、report_path

## 七、执行流程整合

### 7.1 vitest闭环内部检查流程（节点2）

```
执行子代理内部 vitest 闭环：

1. Crush生成Vue/TS代码
   ↓
2. vitest执行（coverage）
   ├── vitest通过 + coverage≥70% → 产出quality_check
   └─ vitest失败 → 进入修复阶段
       ↓
3. 修复阶段（针对性修复）：
   ├── ESLint检查 → `eslint --fix`自动修复
   ├── Prettier格式化 → `prettier --write`自动格式化
   ├── TypeScript类型检查 → 针对性修复
   └─ 重新vitest → 循环最多3次
       ├── 成功 → 产出quality_check
       └─ 超限 → 返回超限结果 → FlowTicket创建问题单
```

### 7.2 Guardian节点3验证流程

> **Guardian节点3验证流程详见**：[项目整体测试方案.md](./项目整体测试方案.md) 第三章3.2节

import { describe, it, expect } from 'vitest'
import { mount } from '@vue/test-utils'
import Button from '@/components/Button.vue'

describe('Button Component', () => {
  it('renders correctly', () => {
    const wrapper = mount(Button, {
      props: { label: 'Click Me' },
    })
    expect(wrapper.text()).toContain('Click Me')
  })
  
  it('emits click event', async () => {
    const wrapper = mount(Button, {
      props: { label: 'Click Me' },
    })
    await wrapper.trigger('click')
    expect(wrapper.emitted('click')).toBeTruthy()
  })
  
  it('disabled state', () => {
    const wrapper = mount(Button, {
      props: { label: 'Click Me', disabled: true },
    })
    expect(wrapper.attributes('disabled')).toBeDefined()
  })
})
