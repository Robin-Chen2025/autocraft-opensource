# L3-E2E 测试方案规范

> AutoCraft L3-E2E测试方案执行规范，V2重构版

**版本：** v2.0
**日期：** 2026-05-13
**作者：** AutoCraft 团队
**核心原则：真实浏览器，梯度设计，断言必须验证数据正确性**

---

## 一、V2核心变化

| 维度 | V1 | V2 |
|------|----|-----|
| 前端L2 | vitest+jsdom | ❌取消，合并到L3 |
| L3结构 | 两层（无头+有头） | S/M/L梯度（单功能→单模块→跨模块）|
| 断言 | 验证页面能渲染 | 必须验证数据正确性 |
| 工具 | Playwright | Playwright（不变）|

---

## 二、L3梯度设计

### 2.1 三级粒度

| 粒度 | 范围 | 示例 | 发现bug预期 |
|------|------|------|------------|
| **L3-S（单功能）** | 一个页面内一个功能点 | 打开列表页→筛选→验证结果数量变化 | 基础交互bug |
| **L3-M（单模块）** | 一个模块的完整流程 | 上传图片→识别→确认→入库 | 流程衔接bug |
| **L3-L（跨模块）** | 多模块串联的用户场景 | 识别图片→入库→分析→查看结果→AI讲解 | 集成bug |

### 2.2 覆盖要求

| 粒度 | 最低要求 |
|------|---------|
| L3-S | 每个页面至少1个S级用例 |
| L3-M | 每个模块至少1个M级用例 |
| L3-L | 至少1个L级用例（跨模块串联）|

## 三、测试方案文档结构（5章节）

| 章节 | 内容 | 必填 |
|------|------|------|
| **1. 测试范围** | 测试页面/功能清单、不测试内容 | ✅ |

| **3. 测试场景清单** | 用例ID、粒度、预设数据、操作、预期结果 | ✅ **核心章节** |
| **4. 测试数据准备** | 数据库初始化脚本 | ✅ |
| **5. 验收标准** | Playwright通过率、S/M/L覆盖 | ✅ |

## 四、测试场景清单格式

### 4.1 三要素强制要求

与后端L2相同，每个测试用例**必须**包含预设数据+操作+预期结果：

```markdown
| 用例ID | 粒度 | 预设数据 | 操作 | 预期结果 |
|--------|------|---------|------|---------|
| TC-E2E-Q-001 | S | 数据库有5道数学题、3道语文题 | 打开题目列表→筛选"数学" | 列表显示5条，筛选标签显示"数学" |
| TC-E2E-REC-001 | M | 无 | 打开识别页→上传图片→确认识别结果 | 列表页出现新题目，状态为"已完成" |
| TC-E2E-FULL-001 | L | 无 | 上传图片→识别→查看→AI对话提问 | AI回复包含该题目的解题思路 |
```

### 4.2 断言质量标准

| 要求 | ✅ 正确断言 | ❌ 降级断言 |
|------|-----------|-----------|
| 列表数量 | `expect(items).toHaveCount(5)` | `expect(page).toBeVisible()` |
| 筛选标签 | `expect(tag).toContainText('数学')` | `expect(tag).toBeVisible()` |
| AI回复 | `expect(reply).not.toBeEmpty()` | `expect(replyArea).toBeVisible()` |
| 状态显示 | `expect(status).toContainText('已完成')` | `expect(status).toBeVisible()` |

**核心：断言必须验证数据内容，不能只验证页面能渲染。**

## 五、执行流程

### 5.1 三步闭环

与后端L2相同：

```
BUILD-TEST: 编写Playwright测试代码
    ├─ 严格按测试方案文档
    ├─ 不运行测试
    └─ 产出：测试代码文件 + 断言报告

TEST-RUN: 运行Playwright测试
    ├─ 启动真实浏览器
    ├─ 如实记录结果
    └─ 产出：测试运行报告

FIX-CODE: 修复bug
    ├─ 根据TEST-RUN发现的code_issue修复
    └─ 修完后重新跑TEST-RUN
```

### 5.2 失败分类

| 分类 | 判断标准 | 处理 |
|------|---------|------|
| code_issue | 前端/后端代码bug | FIX-CODE |
| test_issue | 测试代码有误 | 新BUILD-TEST |
| env_issue | 浏览器/服务环境问题 | 解决环境 |

|------|------|------|

## 七、验收标准

| 指标 | 要求 |
|------|------|
| Playwright通过率 | 100%（SKIP不计入失败）|
| S级覆盖 | 每个页面至少1个S级用例 |
| M级覆盖 | 每个模块至少1个M级用例 |
| L级覆盖 | 至少1个L级用例 |
| 断言精确性 | 必须验证数据内容 |
| AI服务 | 真实调用，超时=SKIP |
| FIX-CODE验收 | 断言数量不减少，严格度不降低 |
