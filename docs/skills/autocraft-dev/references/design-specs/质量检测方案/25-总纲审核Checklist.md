# 总纲审核 Checklist — V2

**版本：** v2.0
**日期：** 2026-05-13
**适用文档：** 测试方案总纲

---

## E系列：阻断项（每项10分）

| ID | 检查内容 | 通过标准 |
|----|---------|---------|
| E01 | 第1章测试范围总览存在 | V2层级定义+触发规则+职责边界完整 |
| E02 | 第2章L1条件触发规范存在 | 3条触发规则+反向规则完整 |
| E03 | 第3章L2/L3测试策略摘要存在 | 后端L2真实DB策略+前端L3梯度设计 |
| E04 | 第4章场景覆盖矩阵存在 | 各层场景数量预估 |
| E05 | 第5章验收标准与执行顺序存在 | 三层验收标准+执行流程+角色分离 |
| E06 | L1触发规则描述完整 | 规则1/2/3+触发关键词+反向规则 |
| E07 | L2核心原则明确 | 真实DB+预设数据+精确验证+AI真实调用 |
| E08 | L3梯度设计完整 | S/M/L定义+覆盖要求 |
| E09 | 角色分离定义明确 | BUILD-TEST/TEST-RUN/FIX-CODE职责+红线 |
| E10 | 废弃层级说明 | 前端L1/L2取消原因 |

## W系列：警告项（每项2分）

| ID | 检查内容 | 通过标准 |
|----|---------|---------|
| W01 | AI服务测试策略 | 正向+异常+超时处理 |
| W02 | 断言报告机制 | 暴露测试质量 |
| W03 | 测试用例格式规范 | 预设数据+操作+预期结果三要素 |
| W04 | 失败分类定义 | code_issue/test_issue/env_issue |
| W05 | FIX-CODE验收标准 | 断言数量不减少+严格度不降低 |

**评分**：总分 = 100 - (E缺失 × 10) - (W缺失 × 2)
