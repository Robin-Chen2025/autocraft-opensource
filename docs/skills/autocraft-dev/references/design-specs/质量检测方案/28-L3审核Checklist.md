# L3审核 Checklist — V2

**版本：** v2.0
**日期：** 2026-05-13
**适用文档：** L3-E2E测试方案

---

## E系列：阻断项（每项10分）

| ID | 检查内容 | 通过标准 |
|----|---------|---------|
| E01 | 5章节结构完整 | 测试范围/环境/场景清单/数据准备/验收标准 |
| E02 | L3梯度设计存在 | S/M/L三级粒度定义 |
| E03 | S级用例覆盖 | 每个页面至少1个S级用例 |
| E04 | M级用例覆盖 | 每个模块至少1个M级用例 |
| E05 | L级用例覆盖 | 至少1个L级跨模块用例 |
| E06 | 测试用例三要素 | 每个用例有预设数据+操作+预期结果 |
| E07 | 预期结果精确 | 断言验证数据内容，不是只验证页面能渲染 |
| E08 | 真实浏览器要求 | 明确Playwright，禁止jsdom/vitest |
| E09 | 验收标准明确 | Playwright通过率100%+S/M/L覆盖 |

## C系列：内容审核（每项5分）

| ID | 检查内容 | 对照文档 |
|----|---------|---------|
| C01 | 页面功能覆盖 | 对照系统功能设计 |
| C02 | 用户流程覆盖 | 对照PRD用户故事 |
| C03 | AI服务场景覆盖 | 真实调用+异常+超时 |
| C04 | 跨模块数据流转验证 | 对照业务流程设计 |
| C05 | 断言质量检查 | 对照断言质量标准表 |

## W系列：警告项（每项2分）

| ID | 检查内容 | 通过标准 |
|----|---------|---------|
| W01 | 测试数据与后端L2一致 | 共用预设数据 |
| W02 | AI超时=SKIP | 不计入失败 |
| W03 | 断言报告格式 | 每个用例断言内容可输出 |
| W04 | 失败分类 | code_issue/test_issue/env_issue |

**评分**：总分 = 100 - (E缺失 × 10) - (C缺失 × 5) - (W缺失 × 2)
