**版本：** v2.0

| 维度 | V1 | V2 |
|------|----|-----|
| L1策略 | 所有模块必写 | 仅3条规则触发时编写 |
| 前端L1 | vitest单元测试 | ❌取消 |

---

## 二、L1触发规则

### 规则1：外部服务异常处理

| 条件 | 判断标准 | 必须覆盖 |
|------|---------|---------|
| 调用外部AI/HTTP服务 | 代码中有agent/requests/httpx | 超时、格式异常、错误码、空值 |

**触发关键词**：`agent`、`requests`、`httpx`、`openai`、`API_KEY`

### 规则2：多状态流转

| 条件 | 判断标准 | 必须覆盖 |
|------|---------|---------|
| 实体有3个以上状态 | status字段≥3种值 | 正常流转、非法跳转、回退、并发 |

**触发关键词**：`status`、`state`、`transition`

### 规则3：数据格式解析

| 条件 | 判断标准 | 必须覆盖 |
|------|---------|---------|
| 解析外部返回的JSON | json.loads+外部数据源 | 字段缺失、类型错误、嵌套变化 |

**触发关键词**：`json.loads`、`parse`+外部调用

### 反向规则：不需要L1

- 纯CRUD → L2
- 纯前端UI → L3
- 数据库迁移 → L2验证
- 简单查询筛选 → L2

---

## 三、执行流程

L1采用**BUILD+RUN合一**模式：

```
扫描代码 → 判断是否命中3条规则
    ├─ 命中 → 编写L1测试 → 运行 → 输出结果
    └─ 不命中 → 跳过L1
```

---

## 四、L1测试方案文档结构

| 章节 | 内容 |
|------|------|
| 1. 触发规则分析 | 哪些代码命中了哪条规则 |
| 2. 异常场景清单 | 每条命中规则的测试场景 |
| 3. Mock策略 | 仅mock外部服务，不mock内部逻辑 |
| 4. 验收标准 | pytest通过率100% |

## 五、验收标准

| 指标 | 要求 |
|------|------|
| 触发规则覆盖 | 每条命中规则的所有场景都有测试 |
| pytest通过率 | 100% |
| Mock范围 | 仅mock外部服务 |
