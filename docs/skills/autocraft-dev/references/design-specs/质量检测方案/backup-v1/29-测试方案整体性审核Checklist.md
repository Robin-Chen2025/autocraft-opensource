# 测试方案整体性审核 Checklist

> 测试方案跨文档一致性审核，对21+22+23+24整体审核，确保上下文一致、逻辑无冲突、无重复冗余

**版本：** v1.1
**日期：** 2026-04-15
**适用文档：**
- 21-测试方案总纲生成规范
- 22-BE-L2测试方案规范
- 23-FE-L2测试方案规范
- 24-L3-E2E测试方案规范
- **30-L1-BE测试方案规范**
- **31-L1-FE测试方案规范**
**审核时机：** 各文档单独审核通过后，进行整体性审核

---

## 使用说明

**审核维度**：

| 维度 | 说明 | 扣分权重 |
|------|------|---------|
| **上下文一致性** | 总纲与各方案数据是否匹配 | 10分/项 |
| **逻辑冲突** | 各方案验收标准/流程是否冲突 | 10分/项 |
| **重复冗余** | 各方案是否有重复描述 | 5分/项 |
| **引用完整性** | 总纲引用的22/23/24是否正确 | 5分/项 |
| **数据依赖一致性** | BE-L2→FE-L2→L3数据依赖是否一致 | 10分/项 |

**评分计算**：
```
总分 = 100 - (一致性缺失数 × 10) - (逻辑冲突数 × 10) - (重复冗余数 × 5) - (引用缺失数 × 5)
```

**评分等级**：
- ≥90分：通过，整体定稿
- 70-89分：需修改，修复一致性/逻辑冲突后重新审核
- <70分：不通过，重新设计测试方案体系

## 九、快速审核流程

```
Step1: 检查X01-X17（上下文一致性）
    ├─ 对照总纲与各方案场景数量

Step2: 检查L01-L15（逻辑冲突）

Step3: 检查R01-R16（重复冗余）
```
