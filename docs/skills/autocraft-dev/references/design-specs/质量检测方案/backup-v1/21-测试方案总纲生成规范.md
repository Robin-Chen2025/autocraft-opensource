## 一、规范总览

### 1.1 规范定位

本规范是测试方案总纲生成的**执行规范**，包含：
- **测试方案总纲生成规范**：总纲文档结构、生成流程、审核标准
- **L1测试执行规范**：pytest闭环机制、验收标准、Guardian检查项（开发计划制定依据）

**规范层级关系**：

| 层级 | 规范类型 | 文档 | 说明 |
|------|---------|------|------|
| **设计层** | 顶层架构 | 项目整体测试方案及其附件 | 测试架构设计 |
| **指导层** | 方法论 | 20-测试方案生成规范 | 定义测试方案应该怎么生成 |
| **执行层** | 操作手册 | **本规范（21）** | 总纲生成执行规范 |
| **执行层** | 操作手册 | 22/23/24规范 | L2/L3测试方案执行规范 |

**开发计划制定引用**：

| 引用内容 | 规范位置 | 用途 |
|---------|---------|------|
| 总纲 | 本规范第2-4章 | 验收标准前置、任务拆解依据 |
| L1测试规范 | 本规范第5章（L1测试执行规范）| pytest闭环机制、Guardian检查项 |
| L2/L3测试方案 | 22/23/24规范 | 具体场景清单 |

### 1.2 总纲定位

**总纲是用户审阅入口**，确保三层测试覆盖完整后再分层生成L2/L3具体方案。

```
设计文档 → 测试方案总纲（AI生成）→ 用户审阅确认覆盖完整
              ↓
        分层生成 BE-L2 → FE-L2 → L3（按总纲指引）
              ↓
        开发计划（引用总纲+L1规范+L2/L3方案）
```

### 1.3 总纲价值

| 价值维度 | 说明 |
|---------|------|
| **用户审阅入口** | 总纲完整版，包含API模块分组、页面清单、业务流程清单，用户确认覆盖范围完整 |
| **L1规范明确** | L1测试执行规范在总纲中明确，开发计划制定有依据 |
| **验收标准前置** | 用户审阅总纲时确认三层验收标准 |
| **任务拆解依据** | 明确哪些任务需要拆解测试任务单、哪些内部闭环 |
| **开发计划输入** | 开发计划引用总纲+L1规范+L2/L3方案 |

### 1.4 总纲与具体方案的关系

| 文档 | 长度 | 内容 | 用户审阅 |
|------|------|------|---------|
| **总纲** | 灵活长度 | L1/L2/L3三层定义 + API模块分组 + 页面清单 + 业务流程清单 + 场景覆盖矩阵 + 验收标准 | ✅ 审阅确认覆盖范围完整 |
| **BE-L2方案** | ~500行 | 具体测试场景清单 + 测试数据准备 | ✅ 审阅具体场景 |
| **FE-L2方案** | ~400行 | 具体测试场景清单 + 测试数据准备 | ✅ 审阅具体场景 |
| **L3方案** | ~300行 | 具体业务流程场景清单 + 测试数据状态 | ✅ 审阅具体场景 |

## 二、总纲章节结构规范

### 2.1 总纲章节结构（6章节）

| 章节 | 内容 | 长度预估 | 对开发计划的价值 |
|------|------|---------|-----------------|
| **1. 测试范围总览** | L1/L2/L3三层测试范围定义 + 任务类型分类 | ~50行 | 用户理解三层分工 |
| **2. L1测试执行规范** | pytest闭环机制 + **L1测试场景清单表格** + 验收标准 + Guardian检查项（**核心前置**）| ~60行 | **开发计划制定依据** |
| **3. L2/L3测试策略摘要** | 各层职责分工、数据依赖关系、场景划分规则 | ~40行 | 避免重复 |
| **4. 场景覆盖矩阵** | 各层场景数量预估表格（BE-L2/FE-L2/L3）| ~40行 | 任务拆解依据 |
| **5. 验收标准汇总** | 三层验收标准表格 + Guardian Block/Warning | ~20行 | 用户审阅确认 |
| **6. 执行顺序依赖** | BE-DEV(L1内部) → BE-L2 → FE-DEV(L1内部) → FE-L2 → L3 + 执行流程图 | ~20行 | 计划制定顺序 |

**总长度**：灵活长度（根据项目规模调整，需包含完整覆盖范围）

### 2.2 关键说明（必须理解）

> **总纲不是传统测试方案！**

| 传统测试方案 | AutoCraft总纲 | 区别 |
|-------------|-----------------|------|
| 包含详细测试用例清单 | ❌ 不包含 | 总纲是精简版，不展开具体场景 |
| 测试用例表格（前置/步骤/预期）| ❌ 不包含 | 测试用例在L2/L3具体方案中 |
| L1测试需要独立文档 | ✅ 需要（测试代码清单）| L1生成独立测试方案，见30/31规范 |
| 验收标准是测试通过率 | ✅ pytest通过率100% + 覆盖率≥80% | Guardian Block级别 |

**第5章"L1测试执行规范"定义的是pytest闭环机制**：

| 内容项 | 定义 |
|--------|------|
| 测试范围 | 单任务代码质量（100-200行）|
| 测试工具 | pytest（后端）/ vitest（前端）|
| 执行位置 | **执行子代理内部pytest闭环** |
| 验收标准 | pytest通过率100%、覆盖率≥80%、lint检查通过 |
| Guardian检查项 | E4-E7（验证quality_check结果）|
| 生成文档 | ✅ 测试方案（引用30/31规范）|

**L1测试拆分任务单**：BE-DEV/FE-DEV任务拆分002方案→003测试代码→004程序代码，见30/31规范第八章。

## 五、总纲审核标准

### 5.1 审核评分规则

| 评分等级 | 分数范围 | 结论 |
|---------|---------|------|
| **通过** | ≥90分 | 定稿，可进入分层生成 |
| **需修改** | 70-89分 | 修复阻断项后重新审核 |
| **不通过** | <70分 | 重大缺失，重新生成 |

**评分计算**：
```
总分 = 100 - (阻断项缺失数 × 10) - (警告项缺失数 × 2)
```

### 5.2 阻断项清单（E系列 - 必须满足）

- 任务拆解规则表格（BE-DEV/FE-DEV拆解测试先行，L2/L3拆解）
