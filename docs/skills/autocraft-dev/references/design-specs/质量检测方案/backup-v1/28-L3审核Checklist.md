## 使用说明

**评分计算**：
```
总分 = 100 - (阻断项缺失数 × 10) - (内容审核缺失数 × 5) - (警告项缺失数 × 2)
```

**评分等级**：
- ≥90分：通过，定稿
- 70-89分：需修改，修复阻断项+内容审核缺失项后重新审核
- <70分：不通过，重新生成

## 七、快速审核流程

```
Step1: 检查E01-E05（文档结构）
    └─ 缺失任一 → 不通过，重新生成

Step2: 检查E06-E30（内容完整性）
    └─ 统计缺失数量 → 计算扣分

Step3: 检查C01-C30（内容审核-E2E业务流程模拟验证）
    ├─ 对照业务流程文档验证流程覆盖
    ├─ 对照PRD验证功能覆盖
    ├─ 对照API/UI设计文档验证串联完整
    ├─ 验证两层测试策略完整
    └─ 验证L2依赖明确
    └─ 统计缺失数量 → 计算扣分

Step4: 检查W01-W15（细节完整性）
    └─ 统计缺失数量 → 计算扣分

Step5: 计算总分 → 判断结论

Step6: 填写缺失项清单 → 提供修复建议

Step7: 签字确认
```

**内容审核对照文档清单**：

| 对照文档 | 提取内容 | 对应检查项 |
|---------|---------|-----------|
| 业务流程设计文档 | P0流程清单、分支、异常节点 | C01-C05、C06-C10 |
