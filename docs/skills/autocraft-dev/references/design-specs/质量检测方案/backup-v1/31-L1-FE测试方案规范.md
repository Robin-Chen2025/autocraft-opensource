## 一、概述

本文档是L1前端测试方案的**执行规范**，定义：

- L1-FE测试方案文档结构（测试代码清单）
- vitest闭环机制（执行子代理内部）
- 验收标准与阈值参数
- Guardian检查项（E4-E7）
- execution_log.quality_check结构
- 任务单映射（测试代码生成独立任务单）

> **核心定位**：L1测试解决**代码质量问题**，生成**独立测试方案**（测试代码清单），测试代码生成作为独立任务单排在程序代码生成之前。

## 三、L1测试机制定义

### 2.1 测试范围

| 维度 | 定义 |
|------|------|
| **测试焦点** | 代码质量 |
| **测试范围** | 单任务代码（100-200行）|
| **执行时机** | FE-DEV任务内部vitest闭环 |
| **测试工具** | vitest + @vitest/coverage-v8 |
| **生成文档** | ✅ 测试方案（测试代码清单）|

### 2.2 vitest闭环位置

**核心设计**：vitest闭环在**执行子代理内部**（FlowTicket节点2），不是独立任务单。

```
FE-DEV任务单执行（节点2）：
    │
    ├─ Crush生成代码
    │
    ├─ vitest执行 + 覆盖率采集
    │     ├─ vitest通过 → 继续
    │     └─ vitest失败 → 检查lint → 针对性修复 → vitest → 循环最多3次
    │           └─ 超限 → 返回超限结果 → FlowTicket创建问题单
    │
    ├─ lint检查（修复阶段）
    │     ├─ ESLint无Error → 继续
    │     └─ 有Error → 修复
    │
    ├─ TypeScript类型检查（修复阶段）
    │     ├─ tsc无Error → 继续
    │     └─ 有Error → 修复
    │
    └─ 产出 execution_log.quality_check
          ├─ vitest_passed: true/false
          ├─ coverage_percent: 72
          ├─ eslint_errors: 0
          ├─ tsc_errors: 0
          └─ complexity_score: 5
```

### 2.3 Guardian验证位置

**Guardian节点3验证**（结果验证，不执行检查工具）：

| 检查项ID | 验证内容 | 通过条件 | 不通过处理 |
|---------|----------|---------|-----------|
| E1 | 产出物文件存在 | 所有output_files存在 | 返工节点2（上限2次）|
| E2 | 产出物文件非空 | 所有output_files非空 | 返工节点2（上限2次）|
| E3 | execution_log格式正确 | JSON格式合法 | 返工节点2（上限2次）|
| **E4** | vitest执行通过 | quality_check.vitest_passed=true | 返工节点2（上限2次）|
| **E5** | 覆盖率达标 | quality_check.coverage_percent≥70% | 返工节点2（上限2次）|
| **E6** | lint检查通过 | quality_check.eslint_errors=0 | 返工节点2（上限2次）|
| **E7** | 类型检查通过 | quality_check.tsc_errors=0 | Warning/返工 |

> **核心设计**：Guardian验证execution_log.quality_check字段，不直接执行vitest/eslint/tsc

## 三、验收标准与阈值参数

### 3.1 验收标准表格

| 指标 | 要求 | 检测位置 | Guardian验证 | 适用任务 |
|------|------|----------|-------------|---------|
| **vitest通过率** | 100% | vitest闭环内部 | E4: vitest_passed=true | A类任务 |
| **代码覆盖率** | ≥70% | vitest闭环内部 | E5: coverage_percent≥70% | A类任务 |
| **ESLint** | 无Error | vitest闭环内部（修复阶段）| E6: eslint_errors=0 | A类/B类任务 |
| **TypeScript类型检查** | 无Error | vitest闭环内部（修复阶段）| E7: tsc_errors=0 | A类任务 |

### 3.2 阈值参数说明

| 参数 | 前端阈值 | 说明 |
|------|---------|------|
| coverage_percent | ≥70% | vitest coverage-v8采集，行覆盖率（前端低于后端）|
| eslint_errors | =0 | ESLint检查，Error级别 |
| tsc_errors | =0 | TypeScript类型检查 |
| complexity_score | ≤10 | 组件复杂度评分 |

### 3.3 B类任务处理

| 任务类型 | vitest闭环 | quality_check字段 | Guardian检查 |
|---------|-----------|-----------------|-------------|
| **A类**（业务逻辑代码）| ✅ 需要 | 有值 | E4-E7全部检查 |
| **B类**（基础设施代码）| ❌ 不需要 | null | 跳过E4-E7 |
| **DOC类**（文档生成）| ❌ 不需要 | null | 跳过E4-E7 |

## 四、execution_log.quality_check结构

### 4.1 结构定义

```json
{
  "vitest_passed": true,          // vitest执行结果（true/false）
  "coverage_percent": 72,         // 代码覆盖率百分比（前端≥70%）
  "eslint_errors": 0,             // ESLint Error数量
  "tsc_errors": 0,                // TypeScript类型检查Error数量
  "complexity_score": 5           // 组件复杂度评分
}
```

### 4.2 A类任务示例（通过）

```json
{
  "vitest_passed": true,
  "coverage_percent": 72,
  "eslint_errors": 0,
  "tsc_errors": 0,
  "complexity_score": 5
}
```

**Guardian验证**：E4-E7全部通过 → 任务完成

### 4.3 A类任务示例（覆盖率不足）

```json
{
  "vitest_passed": true,
  "coverage_percent": 55,         // <70% 不达标
  "eslint_errors": 0,
  "tsc_errors": 0,
  "complexity_score": 8           // ≤10 通过
}
```

**Guardian验证**：E5不通过 → 返工节点2

### 4.4 B类任务示例

```json
{
  "vitest_passed": null,
  "coverage_percent": null,
  "eslint_errors": 0,
  "tsc_errors": null,
  "complexity_score": null
}
```

**Guardian验证**：vitest_passed=null → 跳过E4-E7，只检查E1-E3

## 五、测试代码生成规范

### 5.1 测试文件命名

| 规则 | 说明 | 示例 |
|------|------|------|
| 文件位置 | tests/frontend/{module}/ | tests/frontend/components/Button.test.ts |
| 文件命名 | {组件名}.test.ts | TaskList.test.ts |
| 函数命名 | test_{场景描述} | test_task_list_render |

### 5.2 测试代码结构

```typescript
// tests/frontend/components/TaskList.test.ts

import { describe, test, expect } from 'vitest'
import { mount } from '@vue/test-utils'
import TaskList from '@/components/TaskList.vue'

describe('TaskList组件', () => {
  
  test('正向场景：渲染任务列表', () => {
    // 准备测试数据
    const tasks = [
      { id: 1, name: '任务1', status: 'pending' },
      { id: 2, name: '任务2', status: 'completed' }
    ]
    
    // 执行组件挂载
    const wrapper = mount(TaskList, {
      props: { tasks }
    })
    
    // 验证渲染结果
    expect(wrapper.findAll('.task-item').length).toBe(2)
    expect(wrapper.text()).toContain('任务1')
  })
  
  test('异常场景：空列表显示提示', () => {
    const wrapper = mount(TaskList, {
      props: { tasks: [] }
    })
    
    expect(wrapper.find('.empty-tip').exists()).toBe(true)
    expect(wrapper.text()).toContain('暂无任务')
  })
})
```

### 5.3 覆盖率采集配置

```bash
# vitest命令（执行子代理内部）
vitest run tests/frontend/components/TaskList.test.ts \
    --coverage \
    --coverage-reporter=json \
    --coverage-threshold.lines=70
```

**覆盖率报告输出**：`coverage/coverage-final.json`

> **说明**：L1场景清单不在总纲展开，由执行子代理在vitest闭环内部动态生成

### 6.1 场景提取规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 每个组件≥1正向场景 | 基础渲染测试 | test_component_render |

| 每个交互事件≥1场景 | 事件处理测试 | test_component_click |
| 每个状态变化≥1场景 | 状态管理测试 | test_component_state_change |

### 6.2 场景命名规范

```
test_{组件名}_{场景类型}_{描述}

场景类型：
  render    渲染场景
  props     Props边界场景
  event     事件交互场景
  state     状态变化场景

示例：
  test_task_list_render
  test_task_list_props_empty
  test_task_list_event_click
  test_task_list_state_filter
```

### 6.3 最小场景清单示例（单个组件）

## 八、与FlowTicket对接

### 8.1 任务单映射（FE-DEV计划）

| 任务编号 | 变体编码 | 任务内容 | 说明 |
|---------|---------|---------|------|
| 001 | DECODE-FE-DEV-001 | 拆解任务单 | 读取设计文档，生成任务单清单 |
| 002 | DOC-L1-FE-PLAN | 生成L1-FE测试方案 | 本规范定义的测试代码清单 |
| **003** | **BUILD-L1-FE-TEST** | **生成测试代码（独立任务单）** | 根据测试方案生成测试代码文件 |
| 004-N | BUILD-FE-CODE | 生成程序代码 + vitest闭环 | vitest闭环验证测试代码 |
| N+1 | DOC-FE-REPORT | 生成测试报告 | vitest执行结果汇总 |

> **核心设计**：测试代码生成（003）是独立任务单，排在程序代码生成（004）之前

### 8.2 执行流程（FE-DEV计划）

```
FE-DEV计划启动
    │
    ├─ 001: 拆解任务单
    │     输入：UI设计 + 组件规范
    │     输出：任务单清单
    │
    ├─ 002: 生成L1-FE测试方案
    │     输入：UI设计 + 组件规范
    │     输出：docs/test-plans/{plan_id}-L1-FE-test-plan.md
    │     审核：33-L1-FE审核Checklist
    │
    ├─ 003: 生成测试代码（独立任务单）
    │     输入：L1-FE测试方案
    │     输出：tests/frontend/components/*.test.ts
    │     Guardian：测试文件存在 + 非空
    │     **关键**：测试代码在程序代码之前
    │
    ├─ 004-N: 生成程序代码 + vitest闭环
    │     ├─ Crush生成程序代码
    │     ├─ vitest执行（运行测试代码）
    │     ├─ 失败 → 修复 → vitest → 循环最多3次
    │     ├─ ESLint检查 + TypeScript检查
    │     └─ 产出 execution_log.quality_check
    │
    └─ N+1: 生成测试报告
          输出：docs/test-reports/{plan_id}-L1-FE-report.md
          Guardian：final_guardian验收
```

### 8.3 测试先行价值

| 价值维度 | 说明 |
|---------|------|
| **TDD思想** | 测试代码先行，程序代码满足测试要求 |
| **任务独立** | 测试代码生成可追踪、可审阅、可验证 |
| **验收前置** | 测试代码通过后才生成程序代码 |
| **错误隔离** | 测试代码问题不影响程序代码生成 |
