## 使用说明

**评分计算**：
```
总分 = 100 - (阻断项缺失数 × 10) - (警告项缺失数 × 2)
```

**评分等级**：
- ≥90分：通过，定稿
- 70-89分：需修改，修复阻断项后重新审核
- <70分：不通过，重新生成

## 六、快速审核流程

```
Step1: 检查E01-E05（文档结构）
    └─ 缺失任一 → 不通过，重新生成

Step2: 检查E06-E10（测试代码清单-核心）
    └─ 清单缺失 → 任务拆解无依据

Step3: 检查E11-E15（任务单映射）
    └─ 003任务单缺失 → FlowTicket无法调度

Step4: 检查E16-E20（验收标准）
    └─ 验收标准缺失 → Guardian无法验证

Step5: 检查E21-E25（pytest闭环机制）
    └─ pytest闭环位置缺失 → 核心设计缺失

Step6: 检查W01-W10（细节完整性）
    └─ 统计缺失数量 → 计算扣分

Step7: 计算总分 → 判断结论

Step8: 签字确认
```
