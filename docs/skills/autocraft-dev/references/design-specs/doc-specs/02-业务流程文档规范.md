## 一、文档概述

业务流程文档描述用户操作流程和业务规则，是L3-E2E测试的**操作步骤依据**。

**前置依赖**：
- PRD（必须）：提供用户角色、功能需求、验收标准

**核心作用**：
- 明确用户操作路径
- 定义角色职责
- 识别异常处理

**与PRD验收标准的关系**：

| 文档 | 用途 | L3-E2E测试中的角色 |
|------|------|-------------------|
| **PRD验收标准（AC）** | 断言依据 | 判断"执行结果是否符合预期" |
| **业务流程文档** | 操作步骤 | 定义"测试脚本执行哪些步骤" |

> ⚠️ **分工约定**：PRD验收标准面向**结果验证**（如"保存后列表显示新题目"），业务流程文档面向**路径描述**（如"点击新增→填写表单→点击保存"）。两者不重复。

**颗粒度对比示例**：

| 文档 | 操作描述 | 颗粒度 |
|------|----------|--------|
| **PRD验收标准（AC）** | `When 用户填写题目内容并选择知识点，点击保存` | **粗粒度**：描述用户意图，不关注具体UI操作 |
| **业务流程文档** | `1. 点击"新增题目"按钮 → 2. 在内容输入框输入文字 → 3. 点击知识点下拉框选择"代数" → 4. 点击保存按钮` | **细粒度**：展开每个UI交互步骤 |

> 💡 **使用建议**：L3-E2E测试脚本参考业务流程文档的操作步骤，断言判断参考PRD验收标准的预期结果。

## 二、何时需要

| 场景 | 是否需要 |
|------|---------|
| 简单CRUD操作 | ❌ 不需要 |
| 多角色协作流程 | ✅ 需要 |
| 复杂业务规则 | ✅ 需要 |
| 跨系统集成 | ✅ 需要 |
| L3-E2E测试依据 | ✅ 需要 |
| **P0功能支撑** | ✅ **必须有** |

> ⚠️ **强制要求**：PRD中所有P0功能都必须有对应的业务流程文档支撑，否则L3-E2E测试缺少操作步骤依据。

## 四、分阶段设计策略

### 4.1 业务流程分阶段原则

业务流程采用**骨架整体规划 + 分阶段细化**策略：

| 原则 | 说明 |
|------|------|
| **骨架整体规划** | 所有P0/P1/P2业务流程的主流程框架一次性设计 |
| **分阶段细化** | P0详细设计，P1/P2框架设计 |

### 4.2 各优先级设计深度

| 优先级 | 设计深度 | 内容 |
|--------|---------|------|
| **P0** | ✅ 详细设计 | 完整流程图、步骤说明、异常处理、RACI矩阵 |
| **P1** | ⚠️ 框架设计 | 主流程框架 + 关键步骤概要 |
| **P2** | ⚠️ 框架设计 | 仅定义流程存在，主步骤概要 |

### 4.3 为什么骨架需要完整？

| 原因 | 说明 |
|------|------|
| **流程是骨架性设计** | 决定系统功能的边界 |
| **避免遗漏接口** | P0设计时需要知道P1/P2流程存在，避免接口遗漏 |
| **架构一次性规划** | 符合迭代设计原则，避免后续补设计时发现要重构P0 |

### 4.4 设计指导

参见总纲文档《设计阶段文档规范（总纲）》第10.9节"分阶段设计策略汇总"。

## 五、验收标准

| 检查项 | 标准 | 通过 |
|--------|------|------|
| **流程信息完整** | 触发条件、结束条件明确 | ☐ |
| **流程图清晰** | 使用BPMN标准符号，每个决策节点有对应异常流程 | ☐ |
| **角色职责明确** | RACI矩阵完整（R/A/C/I四种角色） | ☐ |
| **步骤可执行** | 每个步骤有输入/处理/输出 | ☐ |
| **异常流程覆盖** | 每个决策节点的异常分支有处理说明 | ☐ |
| **业务规则清晰** | 规则ID唯一，规则描述无歧义 | ☐ |

> 📋 **详细审核Checklist**：参见 [10-业务流程审核Checklist.md](./10-业务流程审核Checklist.md)

## 六、示例：题目录入完整流程

### 流程图

```
开始 → 学生创建题目 → 选择知识点 → 上传图片 → 保存 → 结束
         ↓                              ↓
      [内容为空？]                  [图片过大？]
         ↓                              ↓
      提示错误                      提示错误
```

### 角色职责

| 角色 | 创建题目 | 选择知识点 | 保存题目 |
|------|---------|-----------|---------|
| 学生 | R | R | R |
| 系统 | A | A | A |

### 流程步骤

| 步骤 | 角色 | 输入 | 处理 | 输出 |
|------|------|------|------|------|
| 1. 创建题目 | 学生 | 题目内容 | 输入内容 | 题目数据 |
| 2. 选择知识点 | 学生 | 知识点列表 | 选择关联 | 关联关系 |
| 3. 上传图片 | 学生 | 图片文件 | 上传存储 | 图片URL |
| 4. 保存 | 系统 | 完整数据 | 持久化 | 题目ID |
